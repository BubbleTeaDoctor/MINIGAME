# Art Direction v0.9.3 - 2D Pixel Style

本文件定义《无限制角斗场》后续统一美术方向：2D 像素风。

---

## 1. 总体风格定位

目标风格：
- 俯视角 / 轻斜视角 2D 像素风战棋竞技场
- 读图优先，不追求超复杂写实细节
- 角色辨识度 > 场景装饰密度
- 技能反馈清晰、信息层级明确
- 允许带一点暗黑奇幻 + 魔兽 / DND 灵感，但避免直接照搬现有 IP 的视觉元素

关键词：
- readable
- chunky
- tactical
- arcane
- heroic
- cursed
- high-contrast

推荐最终观感：
- 地图和单位一眼能分清
- 状态、陷阱、黑洞、尖刺、召唤物都能瞬间识别
- 角色小而清楚，技能特效短促有力，不遮挡棋盘信息

---

## 2. 视角与画面规格

### 视角
- 主视角：六角格战棋视角
- 地图为俯视偏斜，约 55°~65° 的伪俯视感觉
- 角色不做纯侧视，也不做完全顶视
- 角色朝向建议：默认朝右下 / 左上双向镜像为主

### 逻辑尺寸建议
因为当前地图 tile size 目标是 48，所以建议：
- 基础逻辑 tile：48 x 48 区域内的六角占位
- 实际绘制 tile sprite：64 x 64 或 72 x 72 更合理，方便做溢出边缘
- 单位逻辑占位：1 格
- 单位 sprite 推荐框：48 x 48 或 64 x 64
- UI 图标建议：16 x 16 / 24 x 24 / 32 x 32 三档

### 显示放大建议
像素风正式上线时应避免 1:1 小尺寸过糊：
- 逻辑分辨率可偏低
- 实际显示时做整数倍缩放
- 浏览器渲染必须使用 nearest-neighbor / pixelated

---

## 3. 色彩方向

### 主调性
整体色彩应该偏：
- 深色竞技场底
- 中高对比角色轮廓
- 技能和状态使用强区分强调色

### 推荐色彩分区
1. 地图底色
- 深灰蓝
- 暗石板
- 低饱和紫灰

2. 近战职业
- 暖金、铁灰、暗红

3. 法系职业
- 蓝、紫、青色高光

4. 神圣职业
- 金白、淡蓝、乳白

5. 邪能 / 诅咒 / 死灵
- 绿、紫黑、骨白、暗青

6. 陷阱 / 危险地形
- 红橙、警示黄、深褐

### 功能色统一建议
- 伤害：红 / 橙红
- 治疗：绿
- 格挡 / 护盾：蓝白
- 点燃：橙黄
- 减速：冰蓝
- 控制：紫蓝
- 标记：亮黄或亮青
- 负面牌 / 炸弹：暗红紫
- 召唤物：骨白 / 亡灵绿

---

## 4. 角色设计语言

### 通用要求
每个职业必须满足：
- 小尺寸下也能一眼分辨 silhouette
- 不靠大量细碎纹理区分职业
- 优先靠轮廓、武器、头部造型、主色区分

### 角色体量建议
- 角色头身比偏 Q 版但不要太幼
- 推荐 2.5~3.5 头身
- 身体块面清楚，武器要夸张化

### 各职业视觉锚点
1. 战士
- 大肩甲 / 双手武器 / 红金铁灰

2. 法师
- 法杖 / 长袍 / 蓝紫光效

3. 盗贼
- 兜帽 / 匕首 / 深色轻甲 / 绿或暗红点缀

4. 剑客
- 细长刀 / 收刀姿态 / 黑白红或藏青白

5. 牧师
- 神圣徽记 / 法典或圣杖 / 白金蓝

6. 萨满
- 图腾 / 元素挂件 / 棕、青、火橙混合

7. 死灵法师
- 骨饰 / 骷髅元素 / 骨白 + 亡灵绿 + 暗紫

8. 木士
- 邪能火焰 / 契约纹样 / 紫绿黑

9. 猎人
- 轻甲 / 弓或弩 / 皮革绿褐 / 标记与陷阱视觉件

---

## 5. 地图与棋盘视觉方向

### 地图目标
地图必须服务玩法，不应过度花哨导致格子边界看不清。

### 六角格表现
建议：
- 每格必须有清楚边界
- 但边线不要太亮，避免压过单位
- 可用地表纹理 + 柔和边缘高光来区分

### 地形类型视觉要求
1. 普通地格
- 暗石地 / 古战场地砖
- 纹理轻

2. 黑洞中心
- 高对比旋涡
- 深紫黑核心
- 边缘有吸引粒子
- 必须一眼知道“危险且有引力”

3. 尖刺地形
- 破地而出的骨刺 / 金属刺 / 黑曜石刺
- 必须比普通地格更高危

4. 陷阱
- 可以默认隐藏
- 被发现或已知陷阱要有明显 icon

---

## 6. 技能特效方向

### 原则
- 像素风特效要短促
- 不遮挡棋盘
- 不做太多粒子雾化
- 一次动作 4~8 帧即可有不错效果

### 特效层级建议
1. 选中 / 高亮层
- 柔和外圈亮边
- 不应遮住格内单位

2. 攻击命中层
- 短促闪白 / 砍击线 / 爆点

3. 法术释放层
- 简短投射物或法阵闪动

4. 状态层
- burn：火星
- slow：冰蓝脚下圈
- mark：头顶准星 / 猎印符号
- shield：蓝白护罩碎片

5. 召唤层
- 死灵 token 激活时可有骨白/绿火短动画

---

## 7. UI 像素风规范

### 手牌
- 卡面边框统一像素化
- 来源标签要清楚：职业 / 武器 / 饰品 / 负面牌
- 双模式卡需要在卡面角标或底部有“Mode / 双模式”标识

### 状态 chip
- 采用 16x16 或 24x24 图标
- 必须高对比、低细节
- 小图标优先识别而非装饰

### 展示区
- 骷髅、骨龙、炸弹、猎印等，都应有独立 icon
- 计数信息需要直接显示数字

### 伤害数字
- 使用像素字体或像素风伪像素字体
- 数值动画尽量垂直弹出，避免遮挡整块棋盘

---

## 8. 动画规格建议

### 单位基础动作
每个职业后期最低建议：
- idle
- move
- attack
- cast
- hit
- death

### 早期 demo 最低动画要求
如果先做低成本版本，至少做：
- idle 2~4 帧
- attack 4~6 帧
- cast 4~6 帧
- hit 2~3 帧

### 召唤物
- 骷髅：idle + attack pulse
- 骨龙：idle + heavier pulse
- 不一定一开始就做完整走路动画

---

## 9. 建议像素制作规范

### 线条
- 少用纯黑描边
- 优先使用比本体更深一级的彩色描边

### 明暗
- 每个 sprite 控制在 3~5 层主明暗即可
- 避免过度 anti-alias

### 轮廓
- 单位的轮廓必须能在缩小后仍然成立

### 材质
- 金属：高光硬
- 布料：层次简单
- 骨头：偏暖白或病白
- 邪能：高饱和绿紫

---

## 10. 当前最推荐的制作顺序

### 第一阶段：功能优先
1. 棋盘 tile
2. 黑洞 tile
3. 尖刺 tile
4. 9 个职业头像/站姿占位 sprite
5. 骷髅 / 骨龙 / 炸弹 / 猎印 icon
6. UI 边框与按钮
7. 基础技能高亮与状态 icon

### 第二阶段：职业辨识
8. 战士、法师、盗贼、死灵法师、猎人优先正式 sprite
9. 基础攻击和施法动作
10. 常见特效

### 第三阶段：完整 polish
11. 全职业完整动作
12. 更丰富地图主题
13. 武器差异外观
14. 更细的命中特效

---

## 11. 当前项目最适合的像素风结论

最适合这款游戏的不是极端复古低色数，也不是超细腻豪华像素。

推荐定位：
- 中等分辨率像素风
- 强轮廓
- 强功能色
- 偏暗黑奇幻竞技场
- UI 清楚、角色清楚、特效简短

一句话总结：
**像素风战棋竞技场，优先可读性，其次风格化，再其次是炫技。**
